          class MyMenuOption{
            [string]$DisplayName
            [scriptblock]$Script

            [string] ToString () {
              return $This.DisplayName
            }
          }

          function New-MenuItem ([string]$DisplayName,[scriptblock]$Script) {
            $MenuItem = [MyMenuOption]::new()
            $MenuItem.DisplayName = $DisplayName
            $MenuItem.Script = $Script
            return $MenuItem
          }

          $Opts = @(
            $(New-MenuItem -DisplayName "- Lister les interfaces et selection de celle a modifier" -Script { ListInt }),

            $(New-MenuItem -DisplayName "- Desactiver le DHCP" -Script { DHCP-OFF }),
            $(Get-MenuSeparator),
            $("-> DNS"),
            $(Get-MenuSeparator),
            $(New-MenuItem -DisplayName "Infos sur ce Script" -Script { Show-infos }),
            $(New-MenuItem -DisplayName "Quitter" -Script { Write-Host "Bye!" })
          )

          $Chosen = Show-Menu -MenuItems $Opts

          & $Chosen.Script

        }

Modification du bashrc pour des alias


dans le dossier personnel de chaque utilisateur linux nous avons un fichier qui s'apelle .bashrc qui est invisible, pour le voir il faut donc utiliser la commande "ls -a". dedans nous pouvons trouver des paramèttres du compte.

Un d'entre eux est très utile, il permet de créer des alias de commandes:

par exemple, la commande "ll" n'existe pas, elle à été crée pour éviter de taper "ls -l"

la syntaxe n'est pas compliquée, il sufit de reproduire de schema :

alias (la commande que vous voullez créer)='la commande a remplacer'


alias s='sudo'

alias shut='s shutdown -h'

alias geoip='s geoiplookup -f /usr/share/GeoIP/GeoLiteCity.dat'

alias resapache='s /etc/init.d/apache2 restart'

alias resmysql='s /etc/init.d/mysql restart'

alias .. ='cd ..'
